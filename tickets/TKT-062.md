## Description

Running `./deploy.sh` (or `./scripts/build-release.sh`, or `--release`) from a
shell that has not exported `DEVELOPER_DIR` fails at the build step:

    --- Building Release configuration ---
    xcode-select: error: tool 'xcodebuild' requires Xcode, but active developer
    directory '/Library/Developer/CommandLineTools' is a command line tools instance

The machine's `xcode-select -p` points at the Command Line Tools (which ship no
`xcodebuild`), and the build scripts never select a full Xcode. It only worked
interactively because the operator exports
`DEVELOPER_DIR=/Applications/Xcode-beta.app/Contents/Developer` by hand first.

## Reproduction Steps (bugs only)
1. `sudo xcode-select -s /Library/Developer/CommandLineTools` (or a fresh shell on
   a machine where CLT is the active dir and `DEVELOPER_DIR` is unset).
2. `./deploy.sh`
3. Build aborts with the `xcodebuild requires Xcode` error above.

## Acceptance Criteria
- [x] `./deploy.sh` builds without the caller pre-exporting `DEVELOPER_DIR`.
- [x] An explicitly set `DEVELOPER_DIR` (or a valid `xcode-select` Xcode) is
      respected, not overridden.
- [x] A clear, actionable error if no full Xcode is installed at all.

## Investigation

Only `scripts/build-release.sh` invokes `xcodebuild` (line ~65); `deploy.sh` and
`ship-deploy.sh` both delegate to it, and `xcodegen` (line ~54) is a standalone
binary that does not need Xcode (it succeeded in the failing run). So a single
guard in `build-release.sh` covers every entry point.

## Proposed Solution

Add an `ensure_xcode` guard at the top of `scripts/build-release.sh`: respect a
valid `DEVELOPER_DIR` / `xcode-select` Xcode (rejecting the CommandLineTools
dir, which has no `xcodebuild`); otherwise locate an installed Xcode via
`mdfind` with `/Applications/Xcode.app` and `/Applications/Xcode-beta.app`
fallbacks, export `DEVELOPER_DIR`, and fail with guidance if none is found.

## Implementation Plan
- [x] Add `ensure_xcode()` + call near the top of `scripts/build-release.sh`.

## Files Changed
- `scripts/build-release.sh` - add `ensure_xcode` developer-dir guard.

## Test Report
- From a shell with `DEVELOPER_DIR` unset and `xcode-select -p` = CLT, the guard
  resolves `/Applications/Xcode-beta.app` via `mdfind` and exports
  `DEVELOPER_DIR` (verified by sourcing the function and printing the result;
  `xcodebuild -version` then succeeds).

## Verification Checklist (for human)
- [ ] `./deploy.sh` from a fresh Terminal (no `DEVELOPER_DIR` export) builds and
      installs cleanly.

## Regression Report
- An already-exported `DEVELOPER_DIR` and an Xcode-selected `xcode-select` are
  both respected unchanged; only the broken CLT-active path is repaired.
