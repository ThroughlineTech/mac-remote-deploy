## Description

Phase 4 of the backend-decoupling plan (docs/backend-decoupling-plan.md). The web
PWA can now authenticate itself end to end -- no more pasting an iOS-minted token
-- and the Connect screen behaves (Enter submits). A new "Pair Browser" action on
the Mac surfaces a short-lived code the browser claims via POST /api/v1/pair.

Depends only on the pairing endpoint, which already exists (PairingRouteHandler).
Can land independently of Phases 1-3 (all merged).

## Acceptance Criteria
- [x] PWA Connect screen is a `<form>` so Enter submits (no longer click-only).
- [x] PWA pairs itself: enter a code -> POST /api/v1/pair with deviceName

- [x] "Pair Browser" is gated on HTTPS being up (serverURL starts with https).
- [x] Build + full host unit suite green (1 known-flaky WS integration test).

## Design

The raw pairing token IS the bearer token. Flow:
1. Mac generates an 8-char hex token, registers its hash as pending
   (`registerPendingToken`), and shows the raw code.
2. Browser user types the code into the PWA Connect screen.
3. PWA POSTs `/api/v1/pair {token: code, deviceName: "Browser"}`.
4. Server validates the hash is pending, saves a `PairedDevice`, returns success.
5. PWA stores the code as `rd_token` and uses it as the Bearer going forward.

This reuses the existing QR pairing machinery; the only new server-visible field
is `deviceName: "Browser"`. No API or shared-package changes were needed.

The Mac "Pair Browser" sheet mirrors the QR sheet's poll-for-completion pattern
(`PairedDeviceStore.device(forTokenHash:)`) but presents a typeable code + the web
URL instead of a QR image, since the browser may be on another machine.

## Gotchas
- Pairing requires HTTPS: HTTPHandler rejects /api/v1/pair on the plain-HTTP
  (:8080) listener. The PWA disables Pair and shows guidance when not on https;
  the Mac action is disabled until serverURL is https. Over Tailscale the URL is
  `https://<host>:8443/app/`.
- Respect the rate limiter: the PWA shows the 429 message verbatim.
- Inline event handlers (`onsubmit`/`onclick`) match the existing PWA style and
  the current CSP (the whole app already uses them).
