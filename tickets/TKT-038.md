## Description

The menu bar popover has two ways to select a project:

1. **The project list** at the top — tappable rows with a small blue checkmark on the selected one.
2. **A "Project:" dropdown picker** below the project list — a standard macOS Picker control duplicating the same selection.

This is redundant and sloppy. The project list IS the selection mechanism — tapping a row selects the project. The dropdown below it does the same thing. Having both confuses the user about which is canonical and wastes vertical space in the popover.

### What to do

**Remove the "Project:" dropdown picker** from `BuildControlsSection`. The project list in `ProjectsListSection` (with its tap-to-select rows) is the primary and more natural selection mechanism.

**Make the checkmark more prominent** so it's obvious which project is selected:
- The current checkmark is a small, light-blue `Image(systemName: "checkmark")` with `.font(.caption)` and `.foregroundColor(.accentColor)` — it's subtle to the point of being invisible, especially on non-Retina displays or with reduced transparency enabled.
- Options to beef it up:
  - Use `checkmark.circle.fill` instead of plain `checkmark` — filled circle is more visible
  - Increase font size from `.caption` to `.subheadline` or `.body`
  - Use a high-contrast color (solid blue or green filled circle, not just accentColor)
  - Add a subtle background highlight (light blue tint) on the selected row, not just a checkmark — so the selection is conveyed by the row itself, not just a tiny icon
  - Bold the selected project name (already done via `.fontWeight(isSelected ? .semibold : .regular)` at [ProjectRowView.swift](RemoteDeploy/Views/MenuBar/ProjectRowView.swift)) — good, but the weight difference between `.regular` and `.semibold` is subtle at `.subheadline` size

### Where the dropdown lives

[RemoteDeploy/Views/MenuBar/BuildControlsSection.swift:15-23](RemoteDeploy/Views/MenuBar/BuildControlsSection.swift#L15-L23):

```swift
if appState.projects.count > 1 {
    Picker("Project:", selection: $appState.selectedProjectID) {
        ForEach(appState.projects) { project in
            Text(project.name).tag(project.id as UUID?)
        }
    }
    .pickerStyle(.menu)
    .font(.subheadline)
}
```

This entire `if` block can be deleted. The `.onChange(of: appState.projects)` normalization added in TKT-025 should stay (it's on the VStack, not the Picker).

### Where the checkmark lives

[RemoteDeploy/Views/MenuBar/ProjectRowView.swift](RemoteDeploy/Views/MenuBar/ProjectRowView.swift):

```swift
if isSelected {
    Image(systemName: "checkmark")
        .font(.caption)
        .foregroundColor(.accentColor)
}
```

- [ ] `.onChange(of: appState.projects)` normalization from TKT-025 stays in place
