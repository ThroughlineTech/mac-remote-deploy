## Description
Each module defines its own error enum (`BuildError`, `APIError`, `ConnectionError`, etc.) with no shared hierarchy. Create `Packages/RemoteDeployShared/Sources/RemoteDeployShared/Error/RemoteDeployError.swift` with a unified `LocalizedError` enum used across both targets. Map module-specific errors into it where useful.

## Acceptance Criteria
- [ ] New `RemoteDeployError` enum in shared package
- [ ] Cases cover: serverStartFailed, buildFailed, pairingFailed, networkError, fileNotFound, validationFailed
- [ ] Each case has `errorDescription`, `failureReason`, `recoverySuggestion`
- [ ] Existing per-module enums either replaced or wrapped
- [ ] iOS + macOS use the same type at error boundaries
