## Description

The web PWA served at `/app/` is functionally complete but visually unstyled and
rough. A finished visual design was produced in a Claude Design project
(`ec7c28b8-...`, file `preview.html`) with a full DESIGN_BRIEF.md. This ticket
applies that design to `RemoteDeploy/Resources/pwa/` without changing behavior:
restyle/restructure the markup in the render* template strings and overhaul
`style.css`, while preserving every `data-*` action hook, all API calls, the
ACTIONS table, and the domain state shape.

The design matches the macOS menu bar app's look: dark-primary, iOS-blue accent,
macOS-native cards/tabs/status dots, adapted to a mobile-first touch layout that
also frames as a centered window on desktop.

## Acceptance Criteria
- [x] `style.css` rewritten as the design system (CSS custom-property tokens for
      color/type/spacing, dark + light + auto themes, safe-area insets).
- [x] Markup in `app.js` render* functions updated to the design structure/classes
      with all `data-*` hooks preserved; ACTIONS / api() / state shape unchanged.
- [x] New same-origin assets bundled (`icon.svg`); `index.html` + `manifest.json`
      updated for icons + theme metadata. No external fonts/icons/CSS/JS (CSP-safe).
- [x] Every screen + state covered: connect/pairing (incl. insecure-HTTP warning),
      tab bar, projects list/cards/empty, build (picker/Build/building/status pill/

      filesystem browser, loading/error/disabled states.
- [x] No build step / no dependencies introduced; plain CSS + vanilla JS only.
- [x] `node --check` clean on every touched .js; host build + test suite green.

## Investigation

The design project delivered a fully-restyled copy of the real `pwa/` files plus
a `preview.html` harness (mock fetch/WebSocket backend + a state toolbar). The
designer's original snapshot of `app.js` / `projectform.js` / `settingsform.js`
under `uploads/remotedeploy-pwa-design/pwa/` is byte-identical to the repo's
current files on this branch, so there is no behavioral drift to reconcile - the
design changes are confined to markup + CSS and are safe to adopt directly.

Diff of the delivered design vs repo current:
- `style.css` - full rewrite (new design system). No behavior.
- `app.js` - markup-only changes plus three additive, display-only behaviors:
  a `renderHeader()` app toolbar, an auto/light/dark theme toggle persisted under
  a new `rd_theme` localStorage key (`applyTheme` / `toggleTheme`), and structured
  build-log rows (`logRow` / `logLevel` / `fmtLogTime`; buildLog entries become
  `{t,text}` objects rendered client-side, never sent to the server and tolerant
  of legacy plain strings). All ACTIONS, fetch paths, WS subscribe logic, and the
  domain state shape are unchanged. The Build button gains a disabled-while-
  building "loading" state, matching the brief.
- `index.html` - adds icon `<link>`s, apple/mobile web-app meta, theme-color.
- `manifest.json` - adds `scope`, real SVG icon set (any + maskable), theme/bg.
- `icon.svg` - new bundled app icon (gradient RD tile), referenced same-origin.
- `projectform.js`, `settingsform.js` - unchanged (rely on restyled CSS classes).
- `sw.js` - unchanged shell logic; CACHE_NAME bumped to purge stale offline copy.

## Proposed Solution

Adopt the delivered design files verbatim for `style.css`, `app.js`, `index.html`,
`manifest.json`, and add `icon.svg`. Bump `sw.js` CACHE_NAME (v3 -> v4) since the
app shell changed. Leave `projectform.js` / `settingsform.js` as-is.
