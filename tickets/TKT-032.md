## Description

Clicking "Setup Guide" in the menu bar popover does nothing on first attempt. However, if the user clicks "Settings..." first (which opens the Settings window), and THEN clicks "Setup Guide," the Setup Guide window opens normally. After that first successful open, subsequent clicks on "Setup Guide" also work — until the app is relaunched, at which point the cycle repeats.

This suggests the Setup Guide window requires some activation state that "Settings..." establishes as a side effect.

### Suspected root cause (from code inspection)

"Setup Guide" uses `openWindow(id: "setup-assistant")` at [UtilitiesSection.swift:22](RemoteDeploy/Views/MenuBar/UtilitiesSection.swift#L22):

```swift
Button(action: { openWindow(id: "setup-assistant") }) {
    Label("Setup Guide", systemImage: "questionmark.circle")
}
```

`openWindow(id:)` is a SwiftUI environment action (`@Environment(\.openWindow)`) that tells the `WindowGroup` or `Window` scene with that ID to open. In a `MenuBarExtra` app with `LSUIElement = true` (no dock icon), SwiftUI windows don't render until the app has been "activated" at least once via `NSApp.activate()` or equivalent.

"Settings..." uses `SettingsLink` with `.simultaneousGesture(TapGesture().onEnded { NSApp.activate() })` at [UtilitiesSection.swift:30-37](RemoteDeploy/Views/MenuBar/UtilitiesSection.swift#L30-L37):

```swift
SettingsLink {
    ...
}
.buttonStyle(.plain)
.simultaneousGesture(TapGesture().onEnded { NSApp.activate() })
```

The `NSApp.activate()` call is what makes the app "activated" in AppKit's window lifecycle. Once activated, subsequent `openWindow(id:)` calls work because the app's window server connection is now warm.

**The fix**: the "Setup Guide" button also needs `NSApp.activate()` before calling `openWindow(id:)`, same as the `SettingsLink` does — OR the button action should call `NSApp.activate()` then dispatch `openWindow` to the next runloop turn:

```swift
Button(action: {
    NSApp.activate()
    openWindow(id: "setup-assistant")
}) { ... }
```

This is the same pattern documented in Apple's `MenuBarExtra` → `Window` interop guidance for `LSUIElement` apps.

## Reproduction Steps
1. Launch RemoteDeploy fresh (cold start, not just re-click the menu bar icon).
2. Open the menu bar popover.
3. Click "Setup Guide" — nothing happens.
4. Click "Settings..." — the Settings window opens.
5. Close the Settings window.
6. Click "Setup Guide" again — it now works.
7. Force-quit and relaunch → "Setup Guide" is broken again until Settings is opened first.
