## TKT-040: Companion app shows no build progress after tapping Build & Deploy

**Branch:** `ticket/tkt-040-build-progress`
**Files changed:** 4 | **Tests added:** 0 (existing 14 pass) | **Risk:** medium

### What changed
Fixed 4 root causes: (1) WebSocket now connects on session restore so build status streams work from normal app launch, (2) BuildManager observes WebSocket latestStatus via Combine for real-time updates, (3) ProjectDetailView shows elapsed timer, build phase text, success banner with install link, and failure banner with error, (4) removed unused throwaway BuildManager instance.

### Acceptance Criteria
- [ ] After tapping "Build & Deploy," the companion app immediately shows a loading/progress indicator
- [ ] Build status updates from the server are displayed in real time
- [ ] On build success, the companion shows a clear success state with an actionable next step
- [ ] On build failure, the companion shows an error message with the failure reason
- [ ] The URL slug page is either removed or made secondary to the build progress view

### Verification Steps
- [ ] Kill and relaunch the companion app (normal restore, not fresh pair) — WebSocket should connect automatically
- [ ] Go to Projects → select a project → tap "Build & Deploy"
- [ ] Observe: spinner + "Building..." + elapsed timer appears immediately (not blank)
- [ ] Wait for build to complete — green success banner with "Open Install Page" link appears
- [ ] Tap "Open Install Page" — navigates to the install URL
- [ ] Trigger a build failure (e.g., bad project path) — red error banner with reason appears
- [ ] Tap "View Build Log" — switches to the Build tab
- [ ] Tap "Cancel" during a build — build cancels

### Regression Checks
- [ ] Build tab still works independently (pick project, build, see log)
- [ ] Fresh pairing flow still works (WebSocket connects on pair too)
