## Description
Failures are silent: server start failure prints to console, settings save errors are swallowed, route handler errors never reach the UI. Add `@Published var error: AppError?` to `AppState` and `ConnectionManager`, surface via `.alert()` modifiers, and add retry buttons for transient operations.

## Acceptance Criteria
- [ ] AppState/ConnectionManager expose error state
- [ ] MenuBarView, SettingsView, SetupAssistantView show alerts on failures
- [ ] iOS views show error banners with retry on API failures
- [ ] Server start failure visible in menu bar (red dot + alert)
- [ ] Settings save failure shows specific reason
- [ ] All async ops wrapped with try/catch + user feedback
