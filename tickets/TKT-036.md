## Description

The General tab in Settings has a single "Launch at Login" checkbox centered in a large empty window. It looks unfinished — like a placeholder page that never got filled in. Screenshot confirms: just the checkbox, massive whitespace above and below.

### Options

**A) Add more settings to General.** Candidates that could live here:
- HTTP port (currently hardcoded to 8080, now configurable per TKT-028 — could be user-facing)
- HTTPS port (currently in the Server tab — could consolidate)
- Tailscale polling interval (currently hardcoded to 30s in AppDelegate — TKT-030 will reduce it, but making it user-configurable is an option)
- Auto-start server on launch (currently always starts if certs are configured)
- Theme / appearance (if ever relevant)
- Reset / clear all data button

**B) Merge General into another tab.** "Launch at Login" could move to the Server tab (it's about server lifecycle). Then delete the General tab entirely. Fewer tabs = less clutter.

**C) Redesign the layout.** Keep General but use a `Form` with grouped sections and descriptive text so it doesn't look empty even with one setting. Add a version/build number display at the bottom (currently only on the About tab). Even one setting can look intentional if the layout gives it visual weight.

**Recommendation:** Option B (merge into Server) or Option C (redesign with visual weight). Option A risks bloating a tab with settings that don't belong together just to fill space.

## Reproduction Steps
1. Open Settings.
2. Click the "General" tab.
3. Observe: a single checkbox in a vast empty window.

## Acceptance Criteria
- [ ] The General tab either (a) has enough content to justify its existence with a professional layout, or (b) its sole setting is merged into another tab and General is removed.
- [ ] "Launch at Login" functionality is preserved regardless of where it lives.
- [ ] Settings window doesn't feel like it has a placeholder tab.
- [ ] `xcodebuild test` passes.
