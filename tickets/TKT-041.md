## Description

The Installs tab in the companion iOS app accumulates entries over time with no way to remove or manage them. After many builds, the list grows indefinitely and becomes unwieldy. There is no swipe-to-delete, no "Clear All" button, and no edit mode.

Users should be able to clean up old install entries they no longer need.

## Acceptance Criteria
- [ ] Users can delete individual install entries (e.g., swipe-to-delete or an edit mode with delete buttons).
- [ ] Users can clear all install entries at once (e.g., "Clear All" button or toolbar action).
- [ ] Deleting an entry removes it from the list immediately with appropriate animation.
- [ ] Confirmation prompt before "Clear All" to prevent accidental data loss.
- [ ] `xcodebuild test` passes.

## Investigation

**Regression Risk: Low**

The Installs tab (`RemoteDeployCompanion/Views/History/InstallHistoryView.swift`) is a read-only `List` that fetches records via `APIClient.getInstalls()` (`GET /api/v1/installs`). There is no delete functionality at any layer:

1. **View layer** (line 36): `List(installs)` has no `.swipeActions`, `.onDelete`, or `EditButton`. No "Clear All" toolbar item exists.
2. **API client** (`RemoteDeployCompanion/Services/APIClient.swift`, lines 92-95): Only `getInstalls(limit:)` exists -- no delete or clear methods.
3. **Server route handler** (`RemoteDeploy/API/Routes/InstallsRouteHandler.swift`): Only has `list(_:)` method for `GET`.
4. **API router** (`RemoteDeploy/API/APIRouter.swift`, lines 190-193): Only routes `GET /api/v1/installs` -- no `DELETE` route.
5. **Protocol** (`RemoteDeploy/Protocols/InstallTracking.swift`): `InstallTracking` defines only `recordInstall(...)` and `recentInstalls(limit:)` -- no delete methods.
6. **Implementation** (`RemoteDeploy/Services/ServerInstallTracker.swift`): `ServerInstallTracker` reads/writes a JSON file (`installs.json`) but has no delete or clear capability.
7. **Mock** (`RemoteDeployTests/Mocks/MockInstallTracker.swift`): Also lacks delete methods.

**Root cause**: Delete/clear functionality was never implemented -- the installs feature was built as append-only tracking.

**Regression risk**: Low. Adding new protocol methods, new API endpoints, and new UI actions are all additive changes. Existing `GET /api/v1/installs` and `recordInstall` remain untouched. The only risk is the `ServerInstallTracker` disk write path, which is already well-isolated in an actor.

## Proposed Solution

Add delete-single and delete-all capabilities across the full stack:

- **Server**: Add `deleteInstall(id:)` and `deleteAllInstalls()` to the `InstallTracking` protocol and `ServerInstallTracker` actor. Add `DELETE /api/v1/installs/:id` and `DELETE /api/v1/installs` endpoints to the route handler and API router.
- **API Client**: Add `deleteInstall(id:)` and `deleteAllInstalls()` methods to `APIClient`.
- **View**: Add swipe-to-delete on individual rows and a "Clear All" toolbar button with a confirmation alert to `InstallHistoryView`.

## Implementation Plan

- [ ] **Step 1** — `RemoteDeploy/Protocols/InstallTracking.swift`: Add two new protocol requirements: `func deleteInstall(id: UUID) async -> Bool` (returns true if found and deleted) and `func deleteAllInstalls() async`.
- [ ] **Step 2** — `RemoteDeploy/Services/ServerInstallTracker.swift`: Implement `deleteInstall(id:)` — read records from disk, filter out the matching ID, write back. Implement `deleteAllInstalls()` — write an empty array to disk.
- [ ] **Step 3** — `RemoteDeployTests/Mocks/MockInstallTracker.swift`: Add `deleteInstall(id:)` and `deleteAllInstalls()` implementations with call-count tracking to match the updated protocol.
- [ ] **Step 4** — `RemoteDeploy/API/Routes/InstallsRouteHandler.swift`: Add `delete(_:installID:) -> APIResponse` method that calls `installTracker.deleteInstall(id:)`, returning 200 on success or 404 if not found. Add `deleteAll(_:) -> APIResponse` that calls `installTracker.deleteAllInstalls()` and returns 200.
- [ ] **Step 5** — `RemoteDeploy/API/APIRouter.swift`: In the installs routing block (~line 190), add: `if path matches /api/v1/installs/:id && method == DELETE` → call `installsHandler.delete(...)`. Add: `if path == /api/v1/installs && method == DELETE` → call `installsHandler.deleteAll(...)`.
- [ ] **Step 6** — `RemoteDeployCompanion/Services/APIClient.swift`: Add `func deleteInstall(id: UUID) async throws` that sends `DELETE /api/v1/installs/:id`. Add `func deleteAllInstalls() async throws` that sends `DELETE /api/v1/installs`.
- [ ] **Step 7** — `RemoteDeployCompanion/Views/History/InstallHistoryView.swift`: Add `.swipeActions` with a destructive "Delete" button on each row that calls `deleteInstall(id:)` and removes the entry from the local `installs` array with animation. Add a toolbar "Clear All" button (trash icon) that shows a `.confirmationDialog` and, on confirmation, calls `deleteAllInstalls()` and clears the local array. Hide the "Clear All" button when the list is empty.
- [ ] **Step 8** — `RemoteDeployTests/API/InstallsRouteHandlerTests.swift`: Add tests for `delete` (success + 404 not-found) and `deleteAll` endpoints.
- [ ] **Step 9** — `RemoteDeployTests/InstallTrackerTests.swift`: Add tests for `deleteInstall(id:)` (deletes correct record, returns false for unknown ID) and `deleteAllInstalls()` (empties the store).
- [ ] **Step 10** — Run `xcodebuild test -project RemoteDeploy.xcodeproj -scheme RemoteDeploy -destination 'platform=macOS'` and verify all tests pass.
