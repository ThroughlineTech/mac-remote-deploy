## Description

A companion that scans a fresh pairing QR cannot connect: it reports it can't
reach the server "over Tailscale or locally," and both QR candidates fail.

Root cause (confirmed live on fubars-mac-mini-2): the HTTPS/WSS listener on :8443
presents a Tailscale-provisioned Let's Encrypt certificate
(`fubars-mac-mini-2.tailf3787.ts.net`). That cert is a 90-day cert issued Mar 31
that **expired Jun 29 00:00:58 UTC** -- the same day pairing was attempted. The
on-disk cert file (`~/Library/Application Support/RemoteDeploy/certs/<host>.crt`)
was written once at setup (Mar 30) and never refreshed.

Two compounding bugs let it expire silently:

1. **Nothing renews the cert.** `CertificateProviding.needsRenewal(certPath:)`
   exists and is correct, but it has **zero callers** -- it is dead code. The
   server's 10s poll (`ServerLifecycle.checkTailscaleStatus`) refreshes Tailscale
   connectivity + hostname but never checks cert expiry. The only path to
   `CertProvisioning.provision()` (which runs `tailscale cert`) is a manual API
   hit from the setup UI. So the cert just aged out.

2. **Even a renewed cert wouldn't load.** `ServerLifecycle.reconcileHTTPS()`
   restarts the listener only when the bound port or cert/key *path* changes.
   A renewal writes the same filename, so reconcile is a no-op -- the code comment
   admits it: "A cert REPLACED at the same path still needs a manual relaunch."

Why both pairing paths fail with an expired cert:
- The companion does strict system-trust TLS validation
  (`CertValidatingSessionDelegate.SecTrustEvaluateWithError`). An expired cert ->
  `.cancelAuthenticationChallenge` -> the Tailscale `https://<host>.ts.net:8443`
  attempt fails.
- The local fallback is `http://<lan-ip>:8080`, but the server rejects pairing
  over plain HTTP by design (`403 "Pairing requires HTTPS"`). Pairing *mandates*
  HTTPS, so there is no working fallback once the only HTTPS cert is expired.

## Reproduction Steps (bugs only)
1. Leave the server running past its cert's notAfter (90 days from provisioning).
2. On the Mac, open Pair Device and show the QR.
3. Scan with the companion. Observe: "Could not reach server ... over Tailscale or
   locally." `curl --cacert ... https://<host>:8443/` reports
   "certificate has expired"; `openssl s_client` reports verify error 10.

## Fix

Host/server-side only (`RemoteDeployServer` + `RemoteDeploy/Services`); no
companion or shared-package changes.

1. **Auto-renewal (wire up the dead `needsRenewal`).** New
   `CertRenewalCoordinator` (extracted so the decision is unit-testable) reads the
   configured cert path, calls `needsRenewal`, and -- if expired or within the
   7-day window -- calls `provision()`. `ServerLifecycle` drives it from a new
   `startCertRenewalPolling()`: first check at startup (so a server that booted
   with an already-expired cert self-heals), then every 12h. The provisioner is
   now shared via `ServiceContainer.certProvisioner` so its in-progress guard
   dedupes the timer path against menu-bar-triggered provisioning.

2. **Reload a cert replaced in place.** `HTTPSConfig` gains a `certFingerprint`
   (SHA-256 of the cert file). `reconcileHTTPS()` now differs when the cert
   contents change even if the path is identical, so the renewal's
   `settingsStore.update()` -> `.settingsDidChange` -> reconcile restarts HTTPS
   with the fresh cert. This also closes the documented "manual relaunch" gap for
   menu-bar re-provisioning. The hostname-poll no-op-write behavior is preserved
   (an unchanged cert file yields an identical fingerprint).

### Notes / decisions
- 12h cadence: the cert is 90-day with a 7-day renewal window, so twice-daily is
  ample headroom while keeping the (synchronous `openssl`) check rare; it runs off
  the main actor.
- Fingerprint over mtime: content hash is the correct "did the cert change" signal
  and is cheap on a ~3KB PEM.
- Immediate unblock for the live incident (not part of this code change): re-run
  `tailscale cert` to the configured path and
  `launchctl kickstart -k gui/$(id -u)/com.remotedeploy.server`.
