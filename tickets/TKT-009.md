## Description
`PUT /api/v1/settings` is a no-op: `settingsUpdater` returns `nil` ([RemoteDeployApp.swift configureAPIRouter](RemoteDeploy/RemoteDeployApp.swift)). Settings sent from the iOS companion or PWA are silently dropped. Implement real updater that applies port, certificate paths, and push notification config to `AppState`, validates inputs, and persists to disk.

## Acceptance Criteria
- [ ] Updater validates: port in 1024-65535, cert paths exist + readable, bundle IDs match regex
- [ ] Returns descriptive error string on invalid input
- [ ] Successful update writes to disk and applies in-memory
- [ ] Server restart picks up changes (or hot-reloads where possible)
- [ ] Tests cover happy path + each validation failure
- [ ] iOS companion settings screen reflects changes after PUT
