## Description
`PUT /api/v1/settings` is a no-op: `settingsUpdater` returns `nil` ([RemoteDeployApp.swift configureAPIRouter](RemoteDeploy/RemoteDeployApp.swift)). Settings sent from the iOS companion or PWA are silently dropped. Implement real updater that applies port, certificate paths, and push notification config to `AppState`, validates inputs, and persists to disk.

## Acceptance Criteria
- [x] Updater validates: port in 1024-65535, cert paths exist + readable, bundle IDs match regex
- [x] Returns descriptive error string on invalid input
- [x] Successful update writes to disk and applies in-memory
- [x] Server restart picks up changes (or hot-reloads where possible)
- [x] Tests cover happy path + each validation failure
- [x] iOS companion settings screen reflects changes after PUT
