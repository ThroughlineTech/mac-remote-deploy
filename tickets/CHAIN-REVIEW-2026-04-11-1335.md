## TKT-047: Companion Build tab freezes when log is collapsed (TKT-046 regression)

**Branch:** `ticket/tkt-047-build-tab-freeze-fix`
**Files changed:** 3 (1 prod, 1 new test, 1 project regen) | **Tests:** 313 host passing; companion build clean; new UI test compiles | **Risk:** low

### What changed

Replaces TKT-046's `DisclosureGroup { BuildLogStreamView().frame(maxHeight: .infinity) }` + conditional trailing `Spacer()` (which kept an invisible greedy `ScrollView` in the layout/hit-test tree even when collapsed, swallowing tab-bar taps and shoving the controls to the screen center) with an **explicit conditional render**:

- A plain `Button` toggle header that's always visible (chevron + magnifying-glass icon + "Build Log" label + line-count badge), wrapped in `withAnimation(.easeInOut(0.2))` so the toggle is animated.
- When `isLogExpanded` is `true`: the log fills remaining space (`BuildLogStreamView().frame(maxHeight: .infinity)`).
- When `isLogExpanded` is `false`: just a `Spacer(minLength: 0)` — **the log view is not in the view tree at all**, so it cannot claim space and cannot hit-test.

TKT-046's other behaviors are preserved untouched: collapse-on-new-build (`onChange(of: buildManager.isBuilding)` `false→true` → `isLogExpanded = false`), the auto-scroll `guard buildManager.isBuilding else { return }`, the snap-to-tail on new build, and the trash-button-clears-log toolbar wiring.

A new accessibility identifier `BuildLogToggle` is on the header button so the new UI test can locate it.

### Acceptance Criteria (copied from ticket)

- [ ] On the Build tab with the Build Log collapsed, the bottom tab bar responds to taps and the user can navigate away to any other tab without delay.
- [ ] On the Build tab with the Build Log collapsed, the project picker and Build & Deploy button sit at their natural position (top of the content area), not pushed to the vertical middle.
- [ ] Expanding the Build Log disclosure still works and the log fills the remaining vertical space below the controls.
- [ ] Collapsing the disclosure again returns to the responsive, top-anchored state.
- [ ] No re-introduction of the original TKT-046 problem (log expanded by default, doom scrolling, auto-scroll past terminal state).
- [ ] No main-thread hang observable when the disclosure is collapsed.

### Verification Steps

**The freeze repro is the primary thing to verify.** Run the companion app from the scratch worktree (see top of file) in a simulator or on a paired physical device:

- [ ] **V1.** Launch the companion app. Tap the **Build** tab.
- [ ] **V2.** Observe the layout: navigation title "Build" at top, then immediately beneath it the project picker ("rejog-ios" or whichever), then the Build & Deploy button, then the Build Log header row (chevron + icon + label + line-count badge if any). **The controls should be anchored to the top, NOT shoved to the vertical center.** This is the layout half of the regression — confirming it's gone.
- [ ] **V3.** With the Build Log still **collapsed**, tap **Projects** in the bottom tab bar. Expected: navigates to Projects tab immediately. **Pre-fix bug: tap was ignored.**
- [ ] **V4.** Tap **Build** to come back. Tap **Installs**. Should navigate. Tap **Build** again. Tap **Settings**. Should navigate.
- [ ] **V5.** Back on the Build tab, tap the **Build Log** header row. The log should expand with a small animation; the log content fills the remaining vertical space below the header.
- [ ] **V6.** With the log **expanded**, tap **Projects** in the tab bar. Should still navigate (the expanded log has a real frame, so this should always have worked, but worth checking).
- [ ] **V7.** Come back to the Build tab. Tap the header row again to **collapse**. The log hides, controls stay top-anchored, and tab-bar interaction stays alive — confirm by tapping another tab.
- [ ] **V8.** Tap **Build & Deploy** on a real successful project. While the build runs, the disclosure should be collapsed (because TKT-046's collapse-on-new-build still fires). Confirm Build status / elapsed timer / button-disabled state still update reactively (TKT-043/044 behavior intact).
- [ ] **V9.** Expand the log mid-build, watch lines stream and auto-scroll. Once "Build Succeeded" appears, auto-scroll should stop (TKT-046's terminal-state guard still works). Manual scroll-up should not be fought.
