## Description

The rejog-ios project (`/Users/plympton/src/_project-rejog/rejog-ios`) no longer builds via `xcodebuild archive`. The build works when opened directly in Xcode. Two categories of errors:

1. **Malformed project file**: Xcode warns that the file reference for `rejog-ios/Services` is a member of multiple groups (`"rejog-ios"` and `""`), indicating a corrupted `.xcodeproj/project.pbxproj`.

2. **Swift compile errors**: Multiple files fail to compile with errors like:
   - `Cannot find 'RejogLog' in scope` (in RejogCamera, RejogWidget, and main app targets)
   - `Cannot find type 'SafetySessionManager' in scope`
   - `Cannot find type 'CaptureConfig' in scope`
   - Various missing type/value references across targets

The project depends on `rejog-ios-shared` (a local Swift package at `/Users/plympton/src/_project-rejog/rejog-ios-shared`) which provides RejogLogger, RejogCapture, RejogSafetySession, and other modules. The shared package targets themselves compile, but the main app targets can't resolve their exports.

This started after a refactor. The project builds in Xcode but fails via the `xcodebuild archive` CLI invocation used by RemoteDeploy.

## Reproduction Steps (bugs only)

1. Run the xcodebuild archive command:
   ```
   xcodebuild archive -project /Users/plympton/src/_project-rejog/rejog-ios/rejog-ios.xcodeproj \
     -scheme rejog-ios \
     -archivePath /tmp/RemoteDeploy/rejog-ios-refactor.xcarchive \
     -destination generic/platform=iOS \
     -configuration Release \
     DEVELOPMENT_TEAM=RDJQ523WP4
   ```
2. Build fails with Swift compile errors across RejogCamera, RejogWidget, and rejog-ios targets.
