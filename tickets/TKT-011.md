## Description
`WebSocketManager` and `WebSocketChannelHandler` are implemented but not wired into the NIO pipeline (dead code per security audit, accepted risk #5). Wire the upgrade handler into [NIODeployServer](RemoteDeploy/Services/NIODeployServer.swift), add bearer-token auth at the upgrade level, write tests for subscribe/broadcast/close, and add iOS companion auto-subscription to `buildlog` after pairing. Add reconnect with exponential backoff in `WebSocketClient.swift`.

- [ ] WebSocket upgrade handler wired into NIO channel pipeline
- [ ] Upgrade rejected without valid bearer token
- [ ] WebSocketHandlerTests cover subscribe/unsubscribe/broadcast/close/ping
- [ ] Integration test: connect, subscribe to buildlog, trigger build, receive frames
- [ ] iOS WebSocketClient reconnects with backoff on drop
- [ ] No regression in security audit findings
