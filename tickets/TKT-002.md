## Investigation

### Executive Summary
RemoteDeploy v2.0 is well-architected and security-hardened. All documented features are implemented; security audit closed with zero open critical/high issues. Protocol-oriented DI, proper concurrency primitives (@MainActor, Sendable, OSAllocatedUnfairLock), and clean separation of concerns. Main gaps: sparse test coverage (~136 test methods, critical API paths untested), inconsistent state management between macOS and iOS, scattered logging (20 `print()` vs 8 `os.Logger`), silent error handling, and a few half-wired endpoints (build history, settings update, websocket).

### 2. Best Practices Findings

**SwiftUI architecture** — Strong file organization; consistent `@EnvironmentObject` DI. Gaps: macOS lacks a `BuildManager` (iOS has one); `MenuBarView` is 300+ lines; missing empty/error states in iOS views; navigation pattern split (`openWindow(id:)` on macOS vs `NavigationStack` on iOS).

**Swift concurrency** — Proper `@MainActor` on AppState/ConnectionManager/ServiceContainer; correct `@unchecked Sendable` on lock-protected types; clever [AppStateBridge](RemoteDeploy/RemoteDeployApp.swift) for cross-thread access from event loop. Gaps: status polling task not cancelled on shutdown; build cancel race condition; WebSocket client has no reconnect.

**SwiftNIO server** — Pipeline correct (SSL → HTTP1 → HTTPHandler), 1 MB body cap, dual-listener, proper graceful shutdown. Gaps: no backpressure/flow control; no connection limits; error responses (404/405/500) miss `Content-Length`; no structured request logging.

**Bonjour / Network** — Works. Gaps: no error UI on browser failure; advertiser tied to server start (no explicit invalidation).

**Keychain** — Correctly uses `kSecAttrAccessibleWhenUnlockedThisDeviceOnly` + `.userPresence`. Gap: no item versioning for future migration.

**Sandboxing/entitlements** — Appropriate for a dev tool: `network.server`, `network.client`, `files.user-selected.read-write`. Sandbox disabled (correct — xcodebuild incompatible).

**Error handling** — Minimal. macOS uses `try?` and prints; failures silent. Route handlers return `APIResponse.error()` but UI doesn't surface them. Settings updater returns `nil` (no-op).

**Logging** — 20 `print()` vs 8 `os.Logger`. No subsystem org. iOS APIClient is the gold standard; rest of macOS lags.

**DI / testability** — 9 protocols + ServiceContainer + Mocks/. Gaps: route handlers built inline in [configureAPIRouter (RemoteDeployApp.swift:332-372)](RemoteDeploy/RemoteDeployApp.swift#L332-L372) with hardcoded closures (e.g., `schemeDetector`); no test doubles for NIO server.

**File organization** — Clean. Gaps: [NIODeployServer.swift](RemoteDeploy/Services/NIODeployServer.swift) is 500+ lines (server + HTTPHandler + response routing all in one); [MenuBarView.swift](RemoteDeploy/Views/MenuBarView.swift) is 300+; SetupAssistantView mixes step logic with composition.

**Shared package** — Models live in [Packages/RemoteDeployShared/](Packages/RemoteDeployShared/) and re-exported via `@_exported import`. No duplication detected; could absorb more (validation, formatters, error types).

## Proposed Solution

Treat this ticket as the **deliverable itself** (the audit report you're reading). The recommendations are spun out as 12 follow-up tickets, prioritized below. No code changes are made under TKT-002 — each follow-up ticket can be approved and shipped independently.

### Refactor Recommendations — UI/UX

**HIGH**
1. **Unify state management** — Convert macOS `AppState` to `@MainActor final class` matching iOS `ConnectionManager`. Add a macOS `BuildManager` mirroring iOS. Use reactive streams for derived state. Files: [RemoteDeployApp.swift](RemoteDeploy/RemoteDeployApp.swift), [MenuBarView.swift](RemoteDeploy/Views/MenuBarView.swift), new `RemoteDeploy/Managers/BuildManager.swift`.
2. **Error surfaces & user feedback** — Add `@Published var error: String?` to AppState/ConnectionManager. Wrap async ops with `.alert()`. Propagate route handler errors to UI. Files: [MenuBarView.swift](RemoteDeploy/Views/MenuBarView.swift), [SettingsView.swift](RemoteDeploy/Views/SettingsView.swift), all route handlers.

**MED**
3. **Setup Assistant UX polish** — Progress bar, completion checkmarks, real-time validation, in-step error messages. Files: [SetupAssistantView.swift](RemoteDeploy/Views/SetupAssistantView.swift) + [SetupSteps/](RemoteDeploy/Views/SetupSteps/).
4. **Decompose MenuBarView** — Split 300+ line file into Header/Projects/Build/Utilities subviews under `RemoteDeploy/Views/MenuBar/`.
5. **Empty + error states in iOS views** — [ProjectListView](RemoteDeployCompanion/Views/Projects/ProjectListView.swift), [InstallHistoryView](RemoteDeployCompanion/Views/History/InstallHistoryView.swift), [BuildControlView](RemoteDeployCompanion/Views/Build/BuildControlView.swift).

**LOW**
6. **iOS UX polish** — Haptics on pair, last-build timestamps, pull-to-refresh, build duration display.

### Refactor Recommendations — Architecture

**HIGH**
7. **Comprehensive route handler tests** — Unit tests for all 20 endpoints. Mock `PairedDeviceStoring`/`ProjectStoring`. New: `APIRouterTests`, `PairingRouteHandlerTests`, `NIODeployServerTests`.
8. **Inject route handler dependencies** — Extract closures into protocols (e.g. `SchemeDetecting`). Use a factory in [configureAPIRouter](RemoteDeploy/RemoteDeployApp.swift#L332-L372).
9. **Structured logging everywhere** — Replace 20 `print()` with `os.Logger(subsystem:category:)` organized as `com.remotedeploy.{server,build,api}`.

**MED**
10. **Consolidate error types** — New `RemoteDeployShared/Error.swift` with unified `RemoteDeployError` hierarchy. Replace per-module enums.
11. **Test + secure WebSocket** — Wire handler into NIO pipeline, add token auth at upgrade, write subscribe/broadcast/close tests.
12. **Streamline ProjectConfig** — Rename `bundleID` → `bundleId`, add validation (bundle ID regex, teamID length, platform enum), API CodingKeys.
13. **Implement build history persistence** — New `BuildHistoryStore` (JSON file), wire into [BuildRouteHandler](RemoteDeploy/API/Routes/BuildRouteHandler.swift) (currently `buildHistoryProvider: { [] }` — empty stub).
14. **Complete settings update endpoint** — Implement real `settingsUpdater` (currently returns nil no-op), validate inputs (port range, cert paths), persist to disk.

**LOW**
15. **Extract NIODeployServer** — Split into `NIODeployServer.swift` / `HTTPHandler.swift` / `NIOResponseGenerator.swift`.
16. **Build cancellation safeguards** — Add `isCancelling` flag, post-terminate status verification, race-condition test.
