### Description

In the iOS companion app's **Projects** tab, the project detail view doesn't reactively update when a build is triggered from inside it. Specifically:

1. Go to **Projects** tab → tap a project → the project detail view appears (Name, Bundle ID, Team, Scheme, etc., with a **Build & Deploy** button at the bottom)
2. Tap **Build & Deploy**
3. Nothing visibly changes on the detail view. No progress indicator, no state change, no "Building…" section
4. Navigate to any other tab (e.g. **Build**, **Settings**) and come back to **Projects** → tap the same project → now the detail view shows the expected **Build Status** section with "Build Succeeded", "Open Install Page", and "View Build Log"

So the build state is fetched/refreshed on view appearance but isn't being **observed reactively** — the detail view either:
- Doesn't subscribe to the WebSocket build-status stream at all (it only does a one-shot fetch in `.onAppear`), or
- Subscribes but doesn't apply the updates to its local `@State` / `@Published` binding, or
- Holds onto a stale snapshot of the project that doesn't get replaced when the store emits an update

### Reproduction Steps

1. Open the companion app, go to **Projects**
2. Tap any project to open the detail view
3. Tap **Build & Deploy** at the bottom of the detail view
4. Observe: nothing changes on the screen
5. Wait for the build to actually complete on the Mac
6. Still nothing changes on the detail view
7. Switch to any other tab (Build/Installs/Settings), then tap **Projects** again, then tap the same project
8. Observe: the **Build Status** section now appears with the completed build's status

### Acceptance Criteria

- [ ] Tapping **Build & Deploy** on the project detail view immediately reflects the "Building…" state (spinner / progress / disabled button) without leaving the view
- [ ] As the build progresses, the detail view updates live (via WebSocket or the same subscription mechanism the Build tab uses)
- [ ] When the build completes, the **Build Status** section appears on the detail view without requiring a tab switch
- [ ] Success, failure, and cancelled states all render correctly in-place
