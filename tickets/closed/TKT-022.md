## Description
After TKT-020 shipped, the iOS companion still prompts for passcode 3 times on every cold launch. Additionally, Face ID is never offered — iOS goes straight to passcode — even on a device with Face ID enrolled and working for other apps.

Two distinct root causes:

### 1. `NSFaceIDUsageDescription` missing from Info.plist
`RemoteDeployCompanion/Info.plist` does not declare `NSFaceIDUsageDescription`. iOS requires this key before Face ID is allowed for the app; without it, LocalAuthentication silently falls back to passcode-only. This has been broken since the `.userPresence` hardening landed in v2.0.1 — no one noticed because passcode fallback "worked."

### 2. TKT-020's migration is fragile; `.userPresence` doesn't share auth context across calls
The single-blob approach is correct in theory, but relying on `.userPresence` alone gives **one auth challenge per `SecItemCopyMatching` call**. Even the legacy migration path triggers a full 3-prompt cycle with no shared authentication state. If `SecItemAdd` silently fails on the new blob, every launch re-runs migration → 3 prompts forever.

The correct pattern is an explicit `LAContext` that authenticates once, then is passed via `kSecUseAuthenticationContext` to each subsequent keychain query.

## Reproduction Steps
1. Install the TKT-020 build on a device with Face ID enrolled
2. Cold launch the app → 3 passcode prompts (not Face ID)
3. Force-quit the app, relaunch → still 3 passcode prompts, Face ID still not offered

## Acceptance Criteria
- [x] `NSFaceIDUsageDescription` added to `RemoteDeployCompanion/Info.plist` via `project.yml`
- [x] `KeychainStore` rewritten to use `LAContext` with explicit `evaluatePolicy` authentication
- [x] Face ID is offered first (passcode fallback) on devices with Face ID enrolled
- [x] Exactly **one** prompt per cold launch (migration and fast-path share the same LAContext)
- [x] In-memory cache: subsequent `load()` calls in the same process return the cached value without touching keychain
- [x] Legacy 3-item migration still works; first launch after upgrade also shows only 1 prompt
- [x] iOS build clean
- [x] All 270 macOS tests still pass (unaffected)
