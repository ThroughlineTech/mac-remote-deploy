## Description
`BuildRouteHandler` is constructed with `buildHistoryProvider: { [] }` ([RemoteDeployApp.swift:362-367](RemoteDeploy/RemoteDeployApp.swift#L362-L367)) — always empty. Add a `BuildHistoryStore` (JSON file under Application Support, mirroring `JSONPairedDeviceStore`) that records each completed build (project ID, timestamp, status, duration, error). Wire into the build engine completion callback and into the route handler.

## Acceptance Criteria
- [x] New `RemoteDeploy/Services/BuildHistoryStore.swift` + `BuildHistoryStoring` protocol
- [x] Records appended on build success/failure/cancel
- [x] Capped at last N (e.g. 100) records
- [x] `GET /api/v1/builds/history` returns real data
- [x] Persists across app restarts
- [x] Unit tests cover append, cap, load, query
