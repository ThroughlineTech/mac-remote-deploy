## Description

Create a native macOS companion app (analogous to the iOS `RemoteDeployCompanion`) that can connect to a RemoteDeploy server, trigger builds (both iOS and macOS targets), monitor live build logs, and download built artifacts.

This would let a developer on one Mac trigger and manage builds on another Mac — useful for dedicated build machines, CI setups, or just keeping build load off the development machine.

The macOS companion should support:
- Connecting to a RemoteDeploy server (Tailscale or local network)
- Triggering iOS builds and getting install URLs
- Triggering macOS builds (TKT-051) and downloading `.app`/`.dmg` artifacts
- Live build log streaming via WebSocket
- Project management (list, select, configure)
- Bonjour discovery of RemoteDeploy servers on the local network


## Acceptance Criteria

- [ ] Native macOS app (SwiftUI) that connects to a RemoteDeploy server
- [ ] Can trigger iOS and macOS builds remotely
- [ ] Live build log streaming via WebSocket
- [ ] Can download macOS build artifacts (`.app`/`.dmg`)
- [ ] Bonjour discovery of servers on local network
- [ ] QR code or manual pairing with the server
- [ ] Shares `RemoteDeployShared` package with the server and iOS companion
