## Description

The Server tab in Settings has the right fields but they're floating in a large empty window with no visual structure. Screenshot shows: Port, Hostname, Certificate, and Private Key fields stacked vertically in the bottom half of the window with massive dead space above them. No section headers, no grouping, no status indicators.

### Issues

1. **Fields sink to the bottom** — the content sits low in the window with ~200px of blank space above it. Looks like a layout bug.
2. **No visual grouping** — Port and Hostname are conceptually "Server Identity" while Certificate and Private Key are "TLS Configuration." They should be in separate `GroupBox` or `Form` sections.
3. **No server status at the top** — the user has to look at the menu bar popover to know if the server is running. The Settings tab should show current status (running/stopped, port, URL) prominently at the top with a start/stop/restart button.
4. **Certificate paths are raw filesystem paths** — `/Users/plympton/Library/Application Support/RemoteDeploy/...` is ugly and truncated. Should show the filename only with a tooltip or secondary line for the full path. Or show a "Generated via Tailscale" badge when the cert was auto-generated.
5. **"Auto-Detect" button for hostname** — good, but no visual feedback when it succeeds. A brief checkmark or "Detected: fubars-mac-mini-2.tailf3787.ts.net" confirmation would help.
6. **No "Save" or "Apply" action** — unclear when changes take effect. Do they apply immediately? On next server restart? A "Restart Server" button after changing port/hostname would make the flow explicit.

### Design direction

Match macOS System Settings (Ventura+) style:
- Top section: server status card (green/red dot, port, URL, start/restart button)
- Middle section: "Server Identity" GroupBox (Port, Hostname with Auto-Detect)
- Bottom section: "TLS Certificate" GroupBox (Cert path, Key path, Browse buttons, "Generate via Tailscale" button)
- Tighter vertical rhythm, fields aligned at the top of the window not sinking to the bottom

This can be bundled with TKT-035 (notifications polish) and TKT-036 (General tab) as a "Settings UI polish pass" if preferred — they're all the same genre of work.

## Acceptance Criteria
- [ ] Server settings fields are grouped into labeled sections (e.g., "Server Identity" and "TLS Certificate")
- [ ] Content is anchored at the top of the window, not floating at the bottom
- [ ] Server running status is visible in the Server tab (not only in the popover)
- [ ] Certificate paths show the filename (not the full raw path) with the full path accessible via tooltip or secondary text
- [ ] No functional regression — port, hostname, cert, and key settings still persist and apply correctly
- [ ] `xcodebuild test` passes
