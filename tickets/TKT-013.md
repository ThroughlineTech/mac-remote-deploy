## Description
[ProjectListView](RemoteDeployCompanion/Views/Projects/ProjectListView.swift), [InstallHistoryView](RemoteDeployCompanion/Views/History/InstallHistoryView.swift), and [BuildControlView](RemoteDeployCompanion/Views/Build/BuildControlView.swift) assume data is present. On API failure or while loading: blank screen. Add loading spinners, empty states, and error views with retry buttons.

## Acceptance Criteria
- [ ] Each view has loading / empty / error / loaded states
- [ ] Retry buttons on error states
- [ ] Use ContentUnavailableView (iOS 17+) for empty states
- [ ] Pull-to-refresh on lists
- [ ] UI tests cover each state via mock APIClient
