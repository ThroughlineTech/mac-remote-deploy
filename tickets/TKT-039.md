## Description

Clicking "+ Add Project..." in the menu bar popover opens the Settings window, but it lands on the **Server** tab (the default tab) — not the **Projects** tab where you'd actually add a project. The user has to manually click the "Projects" tab to get where they intended to go. Sloppy.

Screenshot confirms: "Add Project..." was clicked in the popover → Settings window opened to "Server" tab with port/hostname/cert fields showing.

### Root cause

[RemoteDeploy/Views/MenuBar/ProjectsListSection.swift:33-40](RemoteDeploy/Views/MenuBar/ProjectsListSection.swift#L33-L40):

```swift
SettingsLink {
    Label("Add Project...", systemImage: "plus")
    ...
}
.buttonStyle(.plain)
.simultaneousGesture(TapGesture().onEnded { NSApp.activate() })
```

`SettingsLink` is SwiftUI's built-in control for opening the Settings window. It opens Settings to **whatever tab it was last on** (or the first tab if freshly opened). There's no way to tell `SettingsLink` which tab to select — it's a limitation of SwiftUI's `Settings { }` scene API.

### Fix options

**A) Replace `SettingsLink` with a custom button** that opens the Settings window AND programmatically selects the Projects tab. This requires either:
- An `@AppStorage` or `@Binding` for the selected tab in the Settings `TabView`, which the "Add Project" button sets before opening the window.
- Or `openWindow(id:)` on a separate "Add Project" window (the Setup Assistant's project step, or a standalone add-project sheet).

**B) Navigate to the Setup Assistant's project step instead.** The Setup Assistant already has a project-add flow (Step 3: ProjectSetupStep). Clicking "Add Project..." could open the Setup Assistant directly to the project step, which is arguably a better UX than the Settings Projects tab anyway.

**C) Use `NSApp.sendAction(Selector(("showSettingsWindow:")), to: nil, from: nil)`** + programmatically switch tabs via the TabView selection binding. This is the most direct fix but requires exposing the Settings tab selection as shared state.

**Recommendation:** Option A with a `@SceneStorage` or `@AppStorage("selectedSettingsTab")` binding. The Settings TabView binds its selection to this value. The "Add Project" button sets it to `"projects"` before triggering the Settings window open.

## Reproduction Steps
1. Open the menu bar popover.
2. Click "+ Add Project..."
3. The Settings window opens — to the Server tab, not the Projects tab.
4. Expected: Settings opens directly to the Projects tab.
