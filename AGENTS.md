# RemoteDeploy - Agent Guide

RemoteDeploy is a macOS host app that builds iOS/macOS apps and serves them for
install over Tailscale (HTTPS/WSS) and the local network, plus an iOS companion
app that pairs via QR and triggers/monitors builds. The backend runs as a
headless process; the menu bar app is an API client of it.

## Tickets

This repo tracks tickets in **Plane via the Throughline `build` CLI** (config in
`.build/config.toml`). All ticket work - list, read, create, comment, transition,
close, defer - goes through `build`. See `~/src/AGENTS-build-cli-instructions.md`
for the full verb reference.

Do NOT use any of the following - they are obsolete here:
- local `tickets/*.md` files (no longer the system of record),
- `/ticket-*` slash commands or the `ticket-*` skills,
- the Plane MCP (`mcp__plane__*`) or raw `curl` against Plane.

## Project layout

- `RemoteDeploy/` - macOS host menu-bar app (SwiftUI); also holds shared host
  services (build engine, API routes, providers) compiled into the server target.
- `RemoteDeployServer/` - headless backend process: lifecycle, service container,
  NIO API listener (:8080) + HTTPS install/WS server (:8443), Bonjour.
- `RemoteDeployCompanion/` - iOS companion app (SwiftUI). Read-only for project
  config; it lists projects and triggers builds.
- `Packages/RemoteDeployShared/` - shared Swift package (API DTOs, models like
  `ProjectConfig`, error types) used by host, server, and companion.
- `RemoteDeployTests/` - host/server unit tests (hosted on the RemoteDeployServer
  target; `@testable import RemoteDeployServer`).
- `RemoteDeployIntegrationTests/` - HTTP server integration tests.
- `RemoteDeployCompanionUITests/` - companion UI tests.
- `scripts/` - ship-deploy.sh, build-release.sh, capture-screenshots.sh, etc.
- `LaunchAgent/` - launchd plists (`com.remotedeploy.app`, `com.remotedeploy.server`).
- `project.yml` - XcodeGen spec. The `.xcodeproj` is generated, not hand-edited.

## Build / test / ship

- Test: `xcodebuild test -project RemoteDeploy.xcodeproj -scheme RemoteDeploy -destination 'platform=macOS'`
  (the RemoteDeploy scheme's test action runs RemoteDeployTests + RemoteDeployIntegrationTests, hosted on RemoteDeployServer).
- Build: `xcodebuild build -project RemoteDeploy.xcodeproj -scheme RemoteDeploy -destination 'platform=macOS'`
- Ship/deploy: `scripts/ship-deploy.sh` - a dispatch wrapper that inspects the
  merge commit and only runs the full notarized `scripts/build-release.sh`
  pipeline when host code actually changed (companion-only / docs-only ships skip
  it to save notarization quota). Force with `--force`; preview the decision with
  `--dry-run`. Do not ship without explicit instruction.

## Conventions

- The Xcode project is generated by XcodeGen. After adding, removing, or renaming
  source files, run `xcodegen generate` so the `.xcodeproj` (which globs the
  source dirs from `project.yml`) picks them up before building/testing.
- Commit messages: `TKT-ID: short description` when on a ticket; `topic: short
  description` otherwise. No AI-authored trailers or branding.
- Don't merge, push, or ship without explicit instruction - implementing a fix and
  shipping it are separate, explicitly-requested steps.
